## 変更要約

### Core

- local-first な workspace / case storage と deterministic replay
- node / edge / task / decision / event / evidence の event-log-based mutation flow
- storage validation, event verification, cache rebuild command

### Patch / Sync

- `GraphPatch` の validate / review / apply
- required reference integration としての built-in markdown sync
- markdown から patch artifact を生成する reverse sync
- Quickstart / Manual Acceptance の core + sync flow に対する black-box E2E カバレッジ

### Analysis

- frontier / blocker 導出
- impact / critical-path analysis
- slack / bottlenecks / unblock-set / cycles / components / bridges / cutpoints / fragility

### Worker / Extension Surface

- importer / sink / worker plugin protocol
- built-in shell / local-LLM / code-agent worker bridge

### Read-Only Graph Inspection

- `cg case view` を public な read-only working surface として維持
- 着手可能項目は `!`
- 完了項目は `✓`
- 待機中項目は `→`
- blocked 項目は `✗`
- DAG 上で再出現する node は full subtree を重複表示せず `= ... (shared)` で表現

## 既知の非ブロッカー事項

- 広い TUI / graph-view はまだ exploratory で未凍結
- external sink は optional integration であり release blocker ではない
- raw topology / Betti surface は experimental のまま
- `main` branch protection の required check は現状 `Verify` / `E2E` で、admin bypass は許可している

## Tag 方針

1. このノートに追加修正が入ったら、その release-prep tree を commit する。
2. より新しい release-prep commit を候補にする場合は、記録済み verified commit hash を更新する。
3. `v0.1.0-rc1` を tag する。
4. RC 用の追補修正なしで同じ検証セットを維持できる次の clean tree を `v0.1.0` の対象にする。
